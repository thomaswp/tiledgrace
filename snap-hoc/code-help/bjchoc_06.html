Step 7: Prevent Score from going negative

The way we implemented the Score variable, there's nothing preventing Score from going negative. So we're going to want to add that. Ask yourself -- how can you conditionally do something based on a question you need to ask? For example, how do know to stop eating a hot dog? When there's nothing in your hand, right? So coding this up might be something like: When there's still a hot dog left, take a bite. Now take that same idea and apply it to the updating of Score.

To help you out, we've given you a command called if, that will execute the commands inside it only if the condition between the ( and ) is true. A condition often compares two values, so you might say if (x > y) then do something.
